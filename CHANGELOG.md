# InfoMate 변경 사항 (깃허브 버전 대비)

이 문서는 깃허브에 올라가 있는 초기 버전과 현재 개발 중인 버전의 차이점을 정리합니다.

## 📋 추가된 주요 기능

### 1. 백엔드 API 연동 ✅
**파일**: `src/services/api.js`

- **기능**: Django 백엔드 서버와의 통신 구현
- **특징**:
  - POST 요청을 통한 뉴스 URL 분석
  - 백엔드 응답 형식 자동 감지 및 변환
  - 에러 처리 및 상세 로깅
  - ngrok 지원 (ngrok-skip-browser-warning 헤더)
  - 신뢰도 점수 자동 계산
  - 가짜뉴스 여부 자동 판단

**깃허브 버전**: "향후 개발 계획"에 있던 기능 → **구현 완료**

---

### 2. 분석 기록 관리 시스템 ✅
**파일**: `src/services/historyService.js`

- **기능**: 분석 기록 저장, 조회, 검색, 정렬
- **특징**:
  - 로컬 스토리지 기반 영구 저장
  - 해시 테이블을 사용한 O(1) 빠른 조회
  - 정렬 알고리즘 (퀵 정렬, 머지 정렬)
  - 검색 알고리즘 (이진 탐색, 선형 탐색)
  - 기록 추가, 삭제, 검색, 정렬 기능

**깃허브 버전**: "향후 개발 계획"에 있던 기능 → **구현 완료**

---

### 3. 평가 기록 페이지 ✅
**파일**: `src/components/History.vue`

- **기능**: 사용자가 분석한 뉴스 기록을 확인하는 페이지
- **특징**:
  - 분석 기록 목록 표시
  - 검색 기능 (키워드 기반)
  - 정렬 기능 (날짜, 신뢰도 점수)
  - 기록 삭제 기능
  - 기록 클릭 시 상세 결과 페이지로 이동

**깃허브 버전**: 존재하지 않음 → **새로 추가**

---

### 4. 자료구조 구현 ✅
**파일**: `src/utils/dataStructures.js`

- **구현된 자료구조**:
  - **해시 테이블 (HashTable)**: O(1) 조회/삽입/삭제
    - URL 기반 빠른 기록 조회에 사용
  - **그래프 (Graph)**: DFS/BFS 알고리즘 포함
    - 관련 기사 추천 기능에 사용

**깃허브 버전**: 존재하지 않음 → **새로 추가**

---

### 5. 알고리즘 구현 ✅
**파일**: `src/utils/algorithms.js`

- **구현된 알고리즘**:
  - **정렬 알고리즘**:
    - 퀵 정렬 (Quick Sort): O(n log n) 평균
    - 머지 정렬 (Merge Sort): O(n log n) 최악 보장
  - **탐색 알고리즘**:
    - 이진 탐색 (Binary Search): O(log n)
    - 선형 탐색 (Linear Search): O(n)
  - **문자열 알고리즘**:
    - 문자열 매칭 (String Matching)
    - 키워드 추출 (Keyword Extraction)

**깃허브 버전**: 존재하지 않음 → **새로 추가**

---

### 6. URL 파싱 및 검증 유틸리티 ✅
**파일**: `src/utils/urlParser.js`

  - 뉴스 사이트 여부 확인
  - 문자열 알고리즘을 사용한 URL 파싱

**깃허브 버전**: 존재하지 않음 → **새로 추가**

## 🔧 기술적 개선 사항

### 1. 컴퓨터공학과 교육 과정 활용
- **자료구조**: 해시 테이블, 그래프
- **알고리즘**: 정렬 (퀵, 머지), 탐색 (이진, 선형), 그래프 탐색 (DFS, BFS)
- **문자열 알고리즘**: 문자열 매칭, 키워드 추출
- **시간 복잡도 최적화**: O(1) 해시 테이블 조회, O(log n) 이진 탐색

### 2. 성능 최적화
- 해시 테이블을 사용한 빠른 조회
- 캐싱을 통한 불필요한 API 호출 방지
- 효율적인 정렬 알고리즘 사용

### 3. 코드 품질
- 상세한 주석 (시간 복잡도, 공간 복잡도 포함)
- 모듈화된 구조
- 재사용 가능한 유틸리티 함수

## 🎯 주요 성과

1. **백엔드 연동 완료**: Django 백엔드와의 실제 통신 구현

4. **사용자 경험 개선**: 기록 관리, 관련 기사 추천 등 추가 기능
